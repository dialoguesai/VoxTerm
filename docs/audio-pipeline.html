Processing Flow
1024 → 2×512 Sub-Chunks
Each 1024-sample chunk splits into two 512-sample sub-chunks. Returns max probability across sub-chunks.

State persists across calls (streaming). Reset between sessions. Last 64 samples carry forward as context window. get_speech_segments(): min speech 500ms, min silence 300ms, max segment 6.0s.

05 // P2P
Multi-Source Audio Merge

When peers connect, multiple microphones are merged with energy-weighted mixing and a jitter buffer for time alignment.

p2p audio merge // network topology
PeerAudioBuffer
Sequence-Based Ring Buffer
Each peer: 2.0s ring buffer (int16). Frames indexed by sequence number. Gaps → silence fill. Out-of-order / duplicate dropped. read(sec) → float32 [-1, 1].
Energy-Weighted Mix
Closest Mic Wins
w[i] = sqrt(rms[i]) / ∑sqrt(rms[j])
RMS < 0.003 → zero weight. EMA smoothing α=0.15. Result × 1.2 boost, clipped [-1,1].
Jitter Buffer
60ms Time Alignment
Local chunks buffered 60ms before merge. Absorbs network jitter. Zero-latency when no peers. flush() drains on session end.

Hallucination Filter
5-Layer Defense
1. Script mismatch: reject CJK/Cyrillic when expecting Latin
2. Length: < 2 chars or > 80 words
3. N-gram repeat: any n-gram ≥ 3× AND > 25% of text
4. Clichés: "thanks for watching", "[music]", filler words
5. Dedup: last 5 outputs, consecutive duplicates suppressed
